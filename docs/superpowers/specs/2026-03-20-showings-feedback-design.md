# Showings Feedback (In-App) - Design Spec

## 🎯 Общее описание
Система автоматически напоминает риелтору о прошедших показах через 24 часа, требуя указать, состоялся ли показ и оставить комментарий по клиенту.

## 💾 Архитектура и Данные (Без бэкенда)
Мы отказываемся от серверного cron-планировщика и дополнительных таблиц, чтобы идеально вписаться в архитектуру React + Supabase:
- **Без новых таблиц**: Используется существующая таблица `showings`.
- **"Напоминание" (Reminder)** вычисляется на лету на клиенте: 
  `status === 'planned'` И `(Текущее время - showings.showing_date) > 24 часа`.
- **"Обратная связь" (Feedback)**: Сохраняется в уже существующие поля таблицы `showings`:
  - `status` меняется на `completed` или `failed`.
  - `client_feedback` (понравилось, думает, отказ).
  - `feedback_comment` (текст заметки).

## 🖥 UI / UX Компоненты

### 1. Страница "Показы" (`ShowingsPage.jsx`)
Добавляется новый агрегирующий блок в самом верху страницы:
**"📋 Ожидают подтверждения (X)"**
- В блоке списком выводятся карточки просроченных показов.
- На каждой карточке: "Встреча с {клиент} по объекту {адрес} прошла {вчера в 14:00}".
- Вопрос: **Событие состоялось?**
  - Кнопки: **[✅ Да]** и **[❌ Нет]**.
- Если нажато "Да", раскрывается выбор впечатлений клиента (Понравилось / Думает / Нет) и поле для ввода комментария.
- Кнопка **[Сохранить]**. После сохранения диспетчеризируется `UPDATE_SHOWING`, и карточка мгновенно исчезает из списка "Ожидают подтверждения".

### 2. Главная панель / Дашборд (опционально, в будущем)
Можно добавить бейдж или алерт "У вас X показов ожидают подтверждения", ведущий на страницу показов.

## ⚙️ Состояния (Transitions)
- `planned` + прошло 24 часа ➡️ Попадает в блок "Ожидают подтверждения".
- Риелтор жмет "Да" + Сохранить ➡️ Статус `completed`, показ уходит в историю.
- Риелтор жмет "Нет" + Сохранить ➡️ Статус `failed`, показ уходит в историю.
