# Property Import Extension (Avito/CIAN) - Design Spec

## Обзор
Архитектура состоит из двух изолированных проектов: само Chrome-расширение, которое работает в браузере риелтора как отдельная программа, и доработки самого веб-приложения CRM для приема распарсенных данных.

## 1. Расширение (Chrome Browser Extension)
**Структура расширения:**
Папка `extension/` внутри проекта (или отдельно), содержащая:
- `manifest.json` (права доступа к вкладкам avito.ru, cian.ru и нашему сайту CRM).
- `popup.html` и `popup.js` (маленькое окошко при клике на иконку).
- `content.js` (скрипты инжекта, которые вынимают данные).

**Флоу работы:**
1. Риелтор открывает `avito.ru/...`.
2. Жмет на иконку расширения -> "Импортировать".
3. `content.js` собирает объект (JSON): `title`, `price`, `description`, `rooms`, `area`, `photos` (массив ссылок) и `source_url`.
4. Расширение открывает новую вкладку с CRM по адресу `https://[crm-domain]/properties/new?import=true`.
5. Расширение дожидается загрузки вкладки и отправляет в нее `postMessage` с JSON.

## 2. Адаптация CRM (Web App)
**Форма `/properties/new` (`src/pages/Properties/PropertyFormPage.jsx`):**
1. При загрузке компонент слушает `window.addEventListener('message')`.
2. Получая JSON с `type: 'PROPERTY_IMPORT_DATA'`, форма моментально обновляет свои локальные React-стейты (`setForm(...)`).
3. Визуальный фидбек: выводим зеленое уведомление: "✅ Данные успешно импортированы с Авито. Проверьте правильность и сохраните."

**Сохранение (Доработка БД):**
- Расширяем объект `property`, записывая в него `avito_url`, `cian_url` (если они присутствуют).
- Фотографии: мы сохраним массив прямых URL-адресов картинок с Авито (они кэшируются и живут долго).
- При сохранении риелтором вызывается стандартный `dispatch({ type: 'ADD_PROPERTY' })`.

## Граничные случаи
- **Блокировки:** Скрипт получает доступ к уже загруженному DOM-дереву. Авито не сможет понять, что это парсер, так как скрипт работает внутри легитимной сессии пользователя.
- **Фото:** В первой версии MVP мы не будем перекачивать оригиналы фото к себе в Supabase Storage (чтобы не усложнять расширение), а просто сохраним ссылки на изображения, которые предоставляет Авито.
- **Отсутствующие поля:** Если парсер не нашел этажность или какой-то параметр (разная верстка у разных городов/типов), поле в CRM останется пустым, риелтор заполнит его руками перед тем, как нажать "Сохранить".
