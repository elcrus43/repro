# Паттерн: Ручное разделение кода (Manual Chunks)

## Метаданные
- **Категория**: Архитектура / Производительность
- **Обнаружен**: 2026-04-14
- **Использование**: Всегда

## Описание
Vite настроен на ручное разделение бандла на чанки для оптимизации загрузки. Каждый чанк содержит логически связанные зависимости.

## Конфигурация
```js
// vite.config.js
rollupOptions: {
  output: {
    manualChunks: {
      'vendor': ['react', 'react-dom', 'react-router-dom'],
      'supabase': ['@supabase/supabase-js'],
      'docx': ['docxtemplater', 'pizzip', 'file-saver'],
      'xlsx': ['xlsx']
    }
  }
}
```

## Результаты сборки
- `vendor-CrVIZt-b.js` — 48 KB (16.6 KB gzipped)
- `supabase-D_cHmkvK.js` — 173 KB (44.4 KB gzipped)
- `xlsx-DyOez2JA.js` — 418 KB (139 KB gzipped)
- `docx` — отдельный ленивый чанк (загружается по требованию)

## Где применяется
- `vite.config.js` — конфигурация ручных чанков

## Плюсы и минусы
✅ Уменьшает начальный бандл
✅ Позволяет кешировать редко обновляемые зависимости
✅ Лучшая загрузка параллельными запросами
❌ Требует ручного поддержания при изменении зависимостей

## Альтернативы
- Автоматическое разделение (auto-splitting)
- Динамические импорты (`import()`)
