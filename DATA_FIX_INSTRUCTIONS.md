# 🔧 Инструкции по исправлению проблем с сохранением данных

## Проблема
**Клиенты и показы не сохраняются** — данные пропадают после создания/редактирования.

## Причина
1. **Клиенты**: Форма отправляет массив `phones`, но в БД нет такой колонки (есть только `phone`)
2. **Показы**: При создании отсутствует `realtor_id`, что вызывает ошибку RLS Supabase

## Решение

### ✅ ШАГ 1: Применить миграцию в Supabase

1. Откройте **Supabase Dashboard**: https://supabase.com/dashboard
2. Выберите ваш проект
3. Перейдите в **SQL Editor** (левое меню)
4. Скопируйте содержимое файла: `supabase/migrations/030_fix_clients_and_showings_save_issues.sql`
5. Вставьте в SQL Editor и нажмите **Run**

**Что сделает миграция:**
```sql
✓ Добавит колонку phone_2 в таблицу clients
✓ Убедится что client_types имеет правильный default
✓ Убедится что additional_contacts имеет правильный default
✓ Убедится что passport_details может быть NULL
✓ Добавит realtor_id в showings (если отсутствует)
✓ Создаст индексы для производительности
```

### ✅ ШАГ 3: Проверить исправление

1. **Войдите в приложение**
2. **Создайте нового клиента**:
   - Введите ФИО
   - Добавьте 2 телефона (если нужно)
   - Нажмите "Сохранить"
   - ✅ Клиент должен сохраниться

3. **Создайте новый показ**:
   - Выберите объект
   - Выберите клиента
   - Установите дату
   - Нажмите "Сохранить"
   - ✅ Показ должен сохраниться

4. **Проверьте в Supabase Dashboard**:
   - Table Editor → clients → должны появиться новые записи
   - Table Editor → showings → должны появиться новые записи

## Если что-то пошло не так

### Ошибка: "column 'phone_2' does not exist"
Миграция не была применена. Повторите ШАГ 1.

### Ошибка: "new row violates row-level security policy"
Проверьте что:
- Вы авторизованы в приложении
- RLS политики не были изменены после миграции 024

### Данные всё ещё не сохраняются
1. Откройте **DevTools** (F12) → Console
2. Найдите ошибки с префиксом `[Supabase]`
3. Сообщите текст ошибки для диагностики
